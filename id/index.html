Kebijakan Privasi — Compresser

Terakhir diperbarui: 2026-05-06 · Tanggal berlaku: 2026-05-06

1. Siapa kami

Compresser adalah aplikasi seluler untuk kompresi di perangkat gambar, video, file audio, dan PDF. Privasi adalah hal mendasar: kompresi berjalan secara lokal di perangkat Anda dan file Anda tidak pernah diunggah ke server kami — file tersebut tidak pernah meninggalkan perangkat untuk dibaca, disalin, atau dianalisis oleh pihak ketiga mana pun.

Pengontrol data: Maciej Siemiński (wiraswasta, Polandia) — pengembang Compresser. Kontak: mail@macsiem.dev

Kebijakan Privasi ini menjelaskan data pribadi apa yang Compresser proses, bagaimana, mengapa, dan hak apa yang Anda miliki.

2. Ringkasan sederhana

Compresser berjalan sepenuhnya di perangkat Anda — file Anda tidak pernah diunggah ke server kami. Kami mengintegrasikan Google AdMob untuk iklan banner dan interstisial, Sentry untuk pelaporan kerusakan, dan penagihan Apple / Google untuk pembelian dalam aplikasi. Kami tidak mengumpulkan nama, email, alamat, atau lokasi tepat Anda. Anda memiliki hak GDPR penuh termasuk akses, penghapusan, dan portabilitas — sebagian besar dapat dilaksanakan langsung di aplikasi atau dengan menulis ke mail@macsiem.dev.

3. Data yang kami proses

3.1 Data hanya-lokal

Aplikasi menyimpan secara lokal di perangkat Anda:

  • preferensi pengguna (kualitas kompresi, format default, tema),
  • tanda penyelesaian orientasi,
  • riwayat kompresi (stempel waktu dan statistik per-file — tanpa konten file).

Data ini disimpan sepenuhnya di perangkat Anda dan tidak pernah dikirimkan ke server kami atau pihak ketiga mana pun.

3.3 Data diagnostik (Sentry)

Compresser mengintegrasikan Sentry (sentry.io) untuk pelaporan kerusakan dan kesalahan. Sentry mengumpulkan:

  • pelacakan tumpukan kesalahan (log kerusakan),
  • info perangkat (model, versi OS, lokal),
  • versi aplikasi dan nomor build.

Sentry tidak mengumpulkan konten Anda (entri, file, catatan vaksinasi, foto, arsip), nama pengguna, atau PII lainnya. Data diagnostik disimpan oleh Sentry hingga 90 hari.

3.4 Periklanan (AdMob)

Compresser menampilkan iklan non-invasif di Beranda, Hasil, dan Pengaturan (banner) ditambah interstisial setelah proses kompresi. Iklan disampaikan oleh Google AdMob, yang SDK-nya mengumpulkan:

  • ID Iklan (Android Advertising ID / Apple IDFA), dibagikan dengan Google untuk menayangkan iklan;
  • data diagnostik (model perangkat, lokasi tingkat negara, bahasa).

Mode default adalah iklan non-personalisasi. Di iOS, perintah App Tracking Transparency (ATT) meminta persetujuan Anda sebelum pengenal tingkat pelacakan apa pun digunakan; di Android Anda mengontrol ID Iklan di bawah Pengaturan sistem. Iklan dapat dinonaktifkan secara permanen melalui IAP Seumur Hidup / Premium.

3.5 Pembelian dalam aplikasi (IAP)

Compresser menawarkan fitur berbayar melalui Pembelian Dalam Aplikasi Apple (iOS) dan Penagihan Google Play (Android). Saat Anda melakukan pembelian:

  • Apple / Google memproses semua data kartu dan akun — kami tidak pernah melihat nomor kartu Anda, alamat penagihan, atau informasi akun lengkap Anda;
  • Apple / Google mengembalikan token pembelian / tanda terima transaksi yang Compresser simpan secara lokal untuk memvalidasi hak;
  • pemulihan pembelian mengambil ulang hak Anda yang ada dari Apple / Google.

Produk yang tersedia: pembukaan kunci seumur hidup, kotak tip opsional, langganan premium tahunan.

3.6 Izin perangkat

  • Foto / Media (READ_MEDIA_IMAGES / PHPhotoLibrary): menambah atau memproses gambar dari galeri (opsional kecuali diperlukan untuk fungsi inti aplikasi).
  • File (READ_EXTERNAL_STORAGE / iOS Files): memilih dokumen/arsip (hanya saat Anda memilih file).
  • Notifikasi (POST_NOTIFICATIONS / UNUserNotificationCenter): pengingat dan status operasi (opsional).
  • Getar (VIBRATE): umpan balik UI haptik (opsional).

Setiap izin diminta secara kontekstual. Anda dapat menolak izin apa pun dan tetap menggunakan fitur inti (jika berlaku).

3.7 Apa yang TIDAK kami kumpulkan

  • Nama, email, nomor telepon, alamat Anda (tidak diperlukan akun).
  • Daftar kontak, kalender, SMS, log panggilan.
  • Lokasi tepat.
  • Riwayat penelusuran di luar aplikasi.
  • Data biometrik (Face ID / Touch ID hanya digunakan sebagai kunci tingkat sistem dan tidak pernah meninggalkan perangkat Anda).
  • Data kesehatan, keuangan, atau kartu pembayaran.

4. Mengapa kami memproses data ini (tujuan)

  • Menjalankan aplikasi di perangkat Anda — penyimpanan lokal dan (jika ada) enkripsi memungkinkan data Anda tetap ada di antara sesi.
  • Meningkatkan stabilitas — diagnostik Sentry membantu memperbaiki kerusakan yang memengaruhi penggunaan Anda.
  • Mempertahankan tingkat gratis — AdMob memberikan iklan non-invasif untuk mendanai pengguna yang tidak membeli premium.
  • Memproses pembelian — penagihan Apple / Google memenuhi hak premium yang Anda pilih untuk dibeli.

5. Dasar hukum (GDPR Art. 6)

Untuk pengguna di Wilayah Ekonomi Eropa, Britania Raya, dan Swiss:

  • Pelaksanaan kontrak (GDPR Art. 6(1)(b)) — menjalankan aplikasi, memproses pembelian Anda.
  • Kepentingan sah (GDPR Art. 6(1)(f)) — diagnostik Sentry, pencegahan penipuan/penyalahgunaan, iklan non-personalisasi. Kepentingan kami adalah meningkatkan stabilitas dan mempertahankan tingkat gratis; ini tidak mengesampingkan hak-hak dasar Anda.
  • Persetujuan (GDPR Art. 6(1)(a)) — iklan personalisasi, di mana Anda memberikan persetujuan melalui perintah ATT (iOS) atau preferensi dalam aplikasi. Anda dapat menarik persetujuan kapan saja tanpa memengaruhi pemrosesan sebelumnya.
  • Kewajiban hukum (GDPR Art. 6(1)(c)) — hanya ketika kami harus menanggapi permintaan yang sah dari pihak berwenang.

7. Transfer internasional

Beberapa penerima di atas berlokasi di luar Wilayah Ekonomi Eropa (terutama di Amerika Serikat). Jika terjadi transfer, transfer tersebut dilindungi oleh:

  • keputusan kecukupan Komisi Eropa (EU-US Data Privacy Framework) untuk penyedia yang bersertifikat di bawah Kerangka Kerja, termasuk Google LLC;
  • Klausul Kontrak Standar (SCC) sebagaimana diadopsi oleh Komisi Eropa, dilengkapi dengan perlindungan tambahan jika berlaku;
  • mekanisme transfer yang diterbitkan sendiri oleh penerima (misalnya, komitmen privasi perusahaan yang mengikat dari Apple).

Anda dapat meminta salinan perlindungan yang relevan dengan menulis ke mail@macsiem.dev.

9. Hak Anda (GDPR Art. 15–22)

Anda memiliki hak-hak berikut mengenai data pribadi Anda:

  • Hak akses (GDPR Art. 15) — konfirmasi apakah kami memproses data Anda dan salinannya. Sebagian besar data Anda sudah ada di perangkat Anda; ekspor dari aplikasi atau tulis kepada kami untuk sisanya.
  • Hak untuk perbaikan (GDPR Art. 16) — memperbaiki data yang tidak akurat. Edit data langsung di aplikasi.
  • Hak untuk penghapusan / "hak untuk dilupakan" (GDPR Art. 17) — hapus semua data melalui Pengaturan → "Hapus semua data", atau dengan mencopot pemasangan aplikasi.
  • Hak untuk membatasi pemrosesan (GDPR Art. 18) — meminta penghentian sementara; tulis kepada kami.
  • Hak untuk portabilitas data (GDPR Art. 20) — Pengaturan → "Ekspor data" menghasilkan bundel JSON / Markdown.
  • Hak untuk menolak (GDPR Art. 21) — menolak pemrosesan berdasarkan kepentingan sah, terutama iklan bergaya profil. Nonaktifkan iklan personalisasi di pengaturan aplikasi atau melalui kontrol sistem.
  • Hak untuk tidak menjadi subjek keputusan otomatis (GDPR Art. 22) — Compresser tidak membuat keputusan otomatis dengan efek hukum atau yang signifikan serupa tentang Anda.

Untuk menggunakan hak apa pun, tulis ke mail@macsiem.dev. Kami menanggapi dalam waktu 30 hari (dapat diperpanjang 60 hari untuk permintaan kompleks, dengan pemberitahuan).

10. Hak untuk mengajukan keluhan (GDPR Art. 77)

Jika Anda yakin kami memproses data Anda secara tidak sah, Anda dapat mengajukan keluhan kepada otoritas pengawas. Untuk penduduk Polandia:

Prezes Urzędu Ochrony Danych Osobowych (PUODO) ul. Stawki 2, 00-193 Warszawa, Poland https://uodo.gov.pl

Anda juga dapat mengajukan keluhan kepada otoritas pengawas di tempat tinggal Anda yang biasa, tempat kerja, atau tempat dugaan pelanggaran di negara EEA lainnya.

11. Privasi anak-anak

Compresser adalah tidak ditujukan untuk anak-anak di bawah 13 tahun (di bawah 16 tahun jika berlaku menurut hukum setempat). Kami tidak dengan sengaja mengumpulkan data pribadi dari anak-anak di bawah usia tersebut. Jika Anda yakin seorang anak telah memberikan data pribadi kepada kami tanpa persetujuan orang tua, hubungi kami di mail@macsiem.dev dan kami akan menghapusnya segera.

Bendera Apple / Google "Dirancang untuk Keluarga" / "Usia 5+" tidak diatur pada Compresser — aplikasi ini dinilai untuk audiens umum 13+.

12. Keamanan

  • Operasi hanya-lokal — kompresi berjalan sepenuhnya di memori perangkat; file asli dan terkompresi tidak pernah meninggalkan perangkat Anda.
  • Keamanan transportasi — semua panggilan jaringan (iklan, IAP, diagnostik Sentry) menggunakan HTTPS / TLS 1.2+.
  • Minimalisasi — kami hanya mengumpulkan apa yang diperlukan untuk tujuan yang tercantum.
  • Tidak ada cadangan konten file ke server kami — metadata file (ukuran, format, stempel waktu) dapat disimpan untuk riwayat dalam aplikasi, tetapi konten file tidak.

Tidak ada sistem yang 100% aman; harap perbarui perangkat Anda dan gunakan kunci perangkat yang kuat.

13. Transparansi Pelacakan Aplikasi (iOS) dan ID Iklan

Di iOS, sebelum pengenal tingkat pelacakan apa pun digunakan oleh AdMob, Compresser menampilkan perintah sistem App Tracking Transparency. Jika Anda menolak, Compresser hanya menyajikan iklan non-personalisasi.

Di Android, Anda dapat mengatur ulang atau membatasi ID Iklan Anda di bawah Pengaturan → Google → Iklan. Memilih "Hapus ID iklan" menghentikan AdMob menggunakan pengenal pribadi; iklan non-personalisasi tetap ditampilkan.

14. Pembelian dalam aplikasi

Compresser menawarkan produk dalam aplikasi berikut melalui Apple App Store dan Google Play: pembukaan kunci seumur hidup, kotak tip opsional, langganan premium tahunan.

Semua data pembayaran ditangani oleh Apple Inc. (Akun App Store / iTunes) atau Google LLC (Penagihan Play). Kami tidak pernah melihat nomor kartu Anda, alamat penagihan Anda, atau akun Apple ID / Google lengkap Anda. Apple dan Google hanya mengembalikan token pembelian buram dan metadata hak yang kami gunakan untuk membuka fitur. Langganan dapat dibatalkan kapan saja di Pengaturan iOS → ID Apple → Langganan atau di aplikasi Google Play → Langganan; aturan pembatalan diatur oleh Apple / Google.

15. Perubahan pada kebijakan ini

Kami dapat memperbarui kebijakan ini seiring perubahan aplikasi, SDK pihak ketiga, atau hukum yang berlaku. Perubahan material akan diumumkan dalam aplikasi dan tanggal "Terakhir diperbarui" di bagian atas akan berubah. Melanjutkan penggunaan Compresser setelah perubahan berarti menerima kebijakan yang diperbarui.
